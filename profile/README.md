## Core Crawl Data: What Screaming Frog Analyzes

<a href="#">Screaming Frog Mac</a> collects the following data for every URL it crawls:

### Response Codes

Every URL in the crawl has a HTTP response code that indicates its status. Screaming Frog
categorizes these into groups for quick filtering:

**200 OK**: URLs that returned successfully and are the foundation of a healthy site.

**3xx Redirects**: Pages that redirect to another URL. Redirect chains (A redirects to B
which redirects to C) waste crawl budget and dilute link equity. <a href="#">Screaming Frog
SEO Spider Mac</a> maps complete redirect chains, identifying chain length and the final
destination URL.

**4xx Client Errors**: Broken pages — 404 Not Found being the most common. Broken internal
links harm user experience and crawlability. Screaming Frog shows every broken URL and every
page that links to it, enabling systematic broken link fixing.

**5xx Server Errors**: Server-side errors that indicate hosting or application problems
affecting crawlability and user experience.

**Blocked by robots.txt**: URLs that the crawler encountered but were blocked from crawling
by robots.txt directives.

### Title Tags

Page titles are a primary on-page SEO factor and the text that appears in search results.
Screaming Frog collects the title tag from every crawled URL and flags:

**Missing titles**: Pages without any title tag.

**Duplicate titles**: Multiple pages sharing the same title text — a signal that pages may
be competing for the same search term or that template-generated titles are not uniquely
describing page content.

**Titles over 60 characters**: Titles that exceed typical search result display length,
which may be truncated in search results.

**Titles under 30 characters**: Titles that may be too short to fully describe the page
content and leverage available title space.

### Meta Descriptions

Meta descriptions are the summary text that appears below the page title in search results.
While not a direct ranking factor, well-written meta descriptions improve click-through rate.
<a href="#">Screaming Frog Mac</a> reports missing, duplicate, and over-length meta descriptions
across the entire crawled site.

### Heading Structure (H1, H2)

Page heading tags define content hierarchy and provide structural SEO signals. Screaming Frog
analyzes H1 and H2 tags:

**Missing H1**: Pages without a primary heading — the most important on-page heading element.

**Multiple H1**: Pages with more than one H1 tag, which can dilute the heading signal.

**H1 matches title tag**: Whether the page title and H1 describe the same content.

**H2 structure**: Second-level headings for substructure analysis.

## Advanced Analysis Features

### Broken Links

<a href="#">Screaming Frog for Mac</a> Inlinks tab shows every internal link to any selected URL.
For a broken page (404), the Inlinks tab lists every page on the site that links to the broken URL,
with the anchor text of each link — the complete information needed to fix the broken link.
The "Bulk Export → Response Codes → All → Inlinks" export produces a spreadsheet of every
internal broken link with its source page, destination URL, and anchor text.

### Redirect Auditing

<a href="#">Screaming Frog Mac</a> redirect analysis covers:

**Redirect chains**: Sequential redirects where a URL redirects through multiple hops before
reaching the final destination. Chains longer than two hops consume crawl budget unnecessarily.

**Redirect loops**: Circular redirect patterns where URL A redirects to B which redirects
back to A — these pages are entirely uncrawlable.

**301 vs 302 redirects**: Permanent (301) and temporary (302) redirects affect how link equity
is transferred. Screaming Frog identifies each redirect type, enabling correction of 302s that
should be 301s.

### Canonicals

Canonical tags tell search engines which version of a duplicate or near-duplicate page should
be indexed. <a href="#">Screaming Frog SEO Spider Mac</a> audits:

**Pages without canonical tags**: Self-referencing canonical tags are a best practice that
prevents accidental canonicalization issues.

**Canonical chains**: Canonicals pointing to URLs that themselves have a different canonical.

**Canonical mismatches**: Pages where the canonical points to a URL that returns a non-200
status code.

**Non-indexable canonicals**: Canonical tags pointing to noindexed or blocked URLs.
